DNS vs HTTP

Using DNS is better at first glance because for a DNS query, an IP address for the server is directly sent back. Once you get the IP, you can just head over there. Compared to HTTP, where it receives a redirect response (30x) prompting the client to go to another URL. This process essentially means two HTTP requests for one redirection. So the ultimate answer is that DNS has less overhead and requires only one single query, while HTTP involves headers and two HTTP requests (Two RTTs).
